## Git Workflow

When making changes to this repository:

1. **Commit Early and Often:** After completing a feature or fix, create a commit with a clear, descriptive message
2. **Commit Message Format:** Use concise messages that clearly describe what changed and why:
   - Good: `Add dark mode toggle` or `Fix: Prevent AI from playing during player's turn`
   - Avoid: `Update` or `Fix bug` (too vague)
